# 学术 PPT 生成

## 适用范围

用于把研究材料转成可讲述的演示文稿。优先调用 `deck-builder` MCP 生成结构化 PPT；若用户提供 Markdown 或要求从 Markdown 转 PPT，可调用 `ppt-markdown` MCP。

## 默认结构

1. 封面：题目、作者、单位、日期。
2. 研究背景：问题来源、现实或理论意义。
3. 文献脉络：核心研究线索和代表文献。
4. 研究空白：已有研究不足与本研究切入点。
5. 方法或方案：数据、方法、模型、实验设计。
6. 主要发现或预期贡献。
7. 局限与后续工作。
8. 参考文献。

## 工作流

1. 先确认汇报场景：课堂汇报、组会、开题、答辩、项目汇报或会议展示。
2. 确认页数、时长、语言、风格和是否需要参考文献页。
3. 将材料压缩为每页一个中心观点，避免一页堆叠过多文字。
4. 输出 PPT 大纲并标注每页讲述意图。
5. 若用户要求生成文件，调用 `deck-builder` 或 `ppt-markdown` 生成 `.pptx`。

## 输出格式

```markdown
## PPT 规格
- 场景：
- 时长：
- 页数：
- 风格：

## 幻灯片大纲
1. 标题
   - 核心观点：
   - 页面内容：
   - 讲述提示：
   - 图表建议：

## 生成说明
- 推荐 MCP：
- 输出文件名：
```

## 设计约束

- 界面文案优先中文。
- 每页标题应是观点句，不只是名词短语。
- 图表优先服务论证，不为装饰而添加。
- 参考文献页保留完整引用信息，不只放 URL。
