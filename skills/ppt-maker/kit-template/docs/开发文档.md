# PPT 制作套件开发文档

通用 16:9 幻灯片预览与导出工程。工作流：**用户自备素材 → 编写 `slides.js` → 浏览器预览 → 导出 PPT/PNG**。

## 项目结构

```
ppt-maker-kit/
├── src/config/project.js      # 标题、导出文件名、素材根路径
├── src/data/slides.js         # 当前幻灯片数据
├── src/utils/                 # 导出与离屏栅格化
├── src/styles/export-capture.css
├── public/assets/materials/   # 用户素材（套件仅含 placeholder.png）
├── examples/zhuanlun/         # 转轮示例数据（无 PNG）
└── docs/slides-schema.md      # 数据模型
```

## 配置项（project.js）

| 字段 | 说明 |
|------|------|
| `appTitle` / `appSubtitle` / `appDesc` | 顶栏文案 |
| `footerText` / `footerLeftText` | 画布页脚右/左栏 |
| `exports.editableFileName` 等 | 三种导出默认文件名 |
| `exports.pptxTitle` / `pptxAuthor` / `pptxFooterLine` | 可编辑 PPTX 元数据 |
| `content.assetBase` | 素材 URL 前缀，默认 `/assets/materials/` |

## 三种导出

| 模式 | 按钮 | 输出 | 说明 |
|------|------|------|------|
| 可编辑 | 导出 PowerPoint（可编辑） | `ppt-editable.pptx` | 文本框、图片、指标独立对象 |
| 像素级 | 导出像素级 PPT | `ppt-raster.pptx` | 离屏整页 PNG 铺满 |
| 图片 ZIP | 导出幻灯片图片 (ZIP) | `ppt-slides.zip` | `slide-01-{宽}x{高}.png` |

共用 [`slideRasterPipeline.js`](../src/utils/slideRasterPipeline.js) + [`captureSlides.js`](../src/utils/captureSlides.js) + [`export-capture.css`](../src/styles/export-capture.css)。

### 导出分辨率

| 档位 | 键名 | 分辨率 |
|------|------|--------|
| 1080p | `hd` | 1920×1080 |
| 2K | `qhd` | 2560×1440 |
| **4K（默认）** | `uhd` | 3840×2160 |

预览画布宽约 1024px；离屏按档位放大 DOM 再栅格化。

**字号与预览同比例：** 画布 `font-size: calc(16px × --capture-scale)`，并用 `em` 覆盖 `--type-*`，避免 `clamp(vw)` 按浏览器视口计算。详见 `export-capture.css`。

## CLI

```bash
npm run export:pptx
```

输出至 `output/${projectConfig.exports.editableFileName}`，无需启动 dev server。

## 示例切换

```bash
npm run example:default
npm run example:zhuanlun
```

转轮示例须自行将 PNG 放入 `public/assets/materials/`，见 [`examples/zhuanlun/README.md`](../examples/zhuanlun/README.md)。

## 自备素材流程

1. 将图片复制到 `public/assets/materials/`
2. 在 `src/data/slides.js` 的 `cases[].images` 中填写文件名
3. `npm run serve`，滚动浏览各页以触发懒加载
4. 使用顶栏导出

## 文案触点（通用化时修改）
