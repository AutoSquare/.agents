# PPT 制作套件开发文档

通用 16:9 幻灯片预览与导出工程。工作流：**用户自备素材 → 编写 `slides.js` → 浏览器预览 → 导出 PPT/PNG**。

## 项目结构

```
ppt-maker-kit/
├── src/config/project.js      # 标题、导出文件名、素材根路径
├── src/data/slides.js         # 当前幻灯片数据
├── src/utils/                 # 导出与离屏栅格化
│   ├── captureSlides.js       # html-to-image 离屏渲染
│   ├── layerCaptureBrowser.js # export-layer 打标校验
│   ├── editableLayerPipeline.js
│   ├── exportPptx.js          # 可编辑 PPT（两层）
│   └── exportEditableImages.js
├── src/styles/export-capture.css
├── scripts/inspect-pptx.mjs   # 导出 PPTX 结构检查（维护者）
├── public/assets/materials/   # 用户素材（套件仅含 placeholder.png）
├── examples/zhuanlun/         # 转轮示例数据（无 PNG）
└── docs/slides-schema.md      # 数据模型
```

## 四种导出

| 模式 | 按钮 | 输出 | 说明 |
|------|------|------|------|
| 可编辑 PPT | 导出 PowerPoint（可编辑） | `ppt-editable.pptx` | 浏览器双 pass：**背景整页** + **透明前景整页** 两层全幅叠放；前景可整体移动 |
| 可编辑 ZIP | 导出可编辑图片 (ZIP) | `ppt-editable-layers.zip` | 每页 `slide-XX-background-{宽}x{高}.png` + `slide-XX-foreground-{宽}x{高}.png` |
| 像素级 | 导出像素级 PPT | `ppt-raster.pptx` | 离屏整页 PNG 铺满 |
| 图片 ZIP | 导出幻灯片图片 (ZIP) | `ppt-slides.zip` | `slide-01-{宽}x{高}.png` |

可编辑导出与像素级/ZIP **共用** [`captureSlides.js`](src/utils/captureSlides.js) + [`export-capture.css`](src/styles/export-capture.css) 的 html-to-image 离屏流程，**无需 Playwright 或 dev server API**。

### 可编辑导出（CSS 显式分层 + 双 pass）

| 项 | 说明 |
|----|------|
| 引擎 | `html-to-image`（与像素级导出相同） |
| 分层 class | `export-layer-fg`：用户可见内容；`export-layer-bg`：纯装饰 sibling（mesh 底板、cover 光晕） |
| Pass 1 背景 | `body.export-bg-capture` → `display:none` 全部 `.export-layer-fg` → 截整页 |
| Pass 2 前景 | `body.export-fg-capture` → `display:none` 全部 `.export-layer-bg` → `backgroundColor: transparent` 截整页 |
| 打标校验 | 每页导出前 `assertExportLayerCoverage(canvas)`，未标节点阻塞导出 |
| PPT 结构 | 每页 2 张全幅图：底层 background、顶层 foreground |
| 分辨率 | 与像素级/ZIP **共用顶栏下拉**（1080p / 2K / 4K） |
| 触发 | 预览页按钮，需 `npm run serve` 打开浏览器 |

新增 Layout 或内容块时，须在模板为外层容器添加 `export-layer-fg`；页面级装饰使用独立 `export-layer-bg` sibling（勿与内容共用一个会被 `display:none` 的父节点）。对照表见 [`captureSlideLayers.js`](../src/utils/captureSlideLayers.js) 中 `LAYER_SELECTOR`。

核心模块：

- [`layerCaptureBrowser.js`](../src/utils/layerCaptureBrowser.js) — `assertExportLayerCoverage`
- [`editableLayerPipeline.js`](../src/utils/editableLayerPipeline.js) — CSS 双模式逐页渲染
- [`export-capture.css`](../src/styles/export-capture.css) — `export-bg-capture` / `export-fg-capture`
- [`exportPptx.js`](../src/utils/exportPptx.js) — 组装 pptxgenjs 两层 PPT
- [`exportEditableImages.js`](../src/utils/exportEditableImages.js) — 打包分层 PNG ZIP

### 导出分辨率

| 档位 | 键名 | 分辨率 |
|------|------|--------|
| 1080p | `hd` | 1920×1080 |
| 2K | `qhd` | 2560×1440 |
| **4K（默认）** | `uhd` | 3840×2160 |

预览画布宽约 1024px；离屏时读取**当前预览态实际画布尺寸**，在基准画布上按预览排版，再经 `html-to-image` 的 `pixelRatio` 放大至目标分辨率，并在写出前将 PNG 归一化到目标像素尺寸以避免舍入误差。

## CLI

```bash
npm run serve
```

在浏览器预览页使用顶栏导出按钮。`npm run export:pptx` 仅提示需在浏览器中导出。

导出前建议滚动浏览全部幻灯片以触发懒加载图片。

### 验证步骤

0. 若预览因上次导出已空白，先 **F5 刷新** 再测
1. `npm run serve`，滚动浏览全部 18 页
2. 选 1080p，点击「导出 PowerPoint（可编辑）」→ 导出完成后预览正常；pptx 叠放无双重阴影
3. 点击「导出可编辑图片 (ZIP)」→ 重点核对 slide 5：`slide-05-background` 无卡片白框/阴影；`slide-05-foreground` 四卡+标题完整
4. 抽查 cover(1) / catalog(2) / split browser(6) / statement agent(12) / evidence(15) 背景/前景分层
5. 像素级 PPT、单层 PNG ZIP 导出仍应正常

## 自备素材流程

1. 将图片复制到 `public/assets/materials/`
2. 在 `src/data/slides.js` 的 `image` / `images` 字段中填写文件名
3. 运行 `python scripts/sync-materials.py` 校验引用完整性
4. `npm run serve`，滚动浏览各页以触发懒加载
5. 使用顶栏导出

## 布局与 export-layer 约定

用户副本位于 `ppt-projects/{english-slug}/`。设计令牌由 `ui-ux-pro-max` + `map-design-system.mjs` 写入 `src/styles/design-tokens.css`。

### 版式精修要点

| 问题 | 处理 |
|------|------|
| 封面 Logo 裁切 | 白底/高对比容器；`contain`；`partnerLogo` + `heroLogo` |
| 官网截图发虚 | Split 页 `imageFrame: 'browser'`；截图 fit 框体 |
| 目录 hover 无法导出 | `CatalogLayout.vue` 静态样式，不依赖 hover |
| 留白过多 | 提高叙事密度；features/catalog 双栏 bento |
| 成果材料 | `EvidenceLayout.vue` stack + stats + table |

### 布局组件

- `CoverLayout.vue`：封面；`partnerLogo`、`kicker`
- `CatalogLayout.vue`：目录；`catalogPreview`
- `FeaturesLayout.vue`：卡片网格；`tags`、`flowBar`、`duoPanel`
- `SplitLayout.vue`：图文分栏；`imageFrame`、`badges`
- `StatementLayout.vue`：左文右卡；`brandLogos`
- `EvidenceLayout.vue`：成果 stack + 表格
- `DualCaseLayout.vue`：双栏案例（`layout: 'dual'`，须 `export-layer-fg`）

### 旧副本迁移（templateVersion 0.1.0 → 0.2.0）

已有 `ppt-projects/*` 建议 **re-scaffold** 或手动合并：`layouts/`、导出 utils、`export-capture.css`、`export-core.manifest.json`。详见 ppt-maker Skill `EXPORT.md` 三区契约。

### slides.js 扩展字段

| 字段 | 适用 layout | 说明 |
|------|-------------|------|
| `partnerLogo` | cover | 院校 Logo 文件名 |
| `catalogPreview` | catalog | 右侧 bento 预览项 |
| `tags` | features[] | 卡片底部标签 |
| `flowBar` | features | 底部流程条文案数组 |
| `featureStyle` | features | `'flow'` 使用与双端协同、技术架构页统一的大卡片视觉，卡片头部固定展示图标与编号 |
| `duoPanel` | features | 三段式中枢面板；可配置 `title`、`bullets` 和 `icons` |
| `brandLogos` | statement | 收尾页品牌标识数组，字段为 `image` / `alt` |
| `images` | split | 多图并排（如双软著） |
| `stats` | evidence | 顶部统计卡数组，字段为 `value` / `label` |
| `tableHeaders` / `table` | evidence | 成果、合作或奖项清单 |
| `caption` | evidence | 图片堆叠区域下方说明 |
| `imageFrame` | split | `'browser'` 启用浏览器框 |
| `imageFrameType` | split | `'paper'` / `'none'` |
| `imagePosition` | split | CSS object-position |
| `imageAspect` | split | 媒体框固定宽高比，用于让产品截图框贴近图片 |
| `variant` | split | 特定页面的视觉变体；`policy` 用于十五五政策页的竖向材料收紧展示 |
| `badges` | split | 左栏 pill 标签行 |
| `calloutFullWidth` | split | 将说明条固定到页面底部并横向铺满 |

### 素材校验

将图片放入 `public/assets/materials/`，确保 `slides.js` 中 `image` / `images` 文件名与磁盘一致。可选运行 `python scripts/sync-materials.py`（若副本含该脚本）。
