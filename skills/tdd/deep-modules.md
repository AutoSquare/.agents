# Deep Modules（深模块）

出自《软件设计的哲学》（A Philosophy of Software Design）：

**Deep module** = 小型接口 + 大量实现

```
┌─────────────────────┐
│   Small Interface   │  ← 少量方法，简单参数
├─────────────────────┤
│                     │
│                     │
│  Deep Implementation│  ← 复杂度藏在背后
│                     │
│                     │
└─────────────────────┘
```

**Shallow module**（宜避免）= 大型接口 + 微薄实现（多半穿堂）

```
┌─────────────────────────────────┐
│       Large Interface           │  ← 许多方法，复杂参数
├─────────────────────────────────┤
│  Thin Implementation            │  ← 只是透传
└─────────────────────────────────┘
```

设计接口时自问：

- 能否减少方法数量？
- 能否简化参数？
- 能否把更多复杂度藏起来？
