# Grill Me（结构化拷问）

就计划或设计的每个待决面向用户反复追问，直到决策树每条分支闭合、与用户达成共同理解。沿设计树逐支走下，逐项消解决策依赖；**每个问题附上你的推荐答案及简短理由**。

若能通过探索代码库 / 文档 / 配置得到唯一事实，**先探索**，不把已知项放进问卷。

## 硬约束（全宿主通用）

1. **必须取得用户的显式选择**后再进入实现（写代码、改文件、scaffold 等）。
2. **禁止**未获答案就假设默认并继续；**禁止**用「如果你同意我就…」代替正式 intake。
3. 发出问卷后 **必须停止并等待**用户下一条回复（或结构化工具返回值）。
4. 收到答案后，用 **简短表格复述共识**，再开工。

## 两种提问模式

| 模式 | 何时用 | 形态 |
|------|--------|------|
| **顺序 grill**（**默认**） | 计划、架构、**PPT 视觉定稿**、多 token 定稿 | **多轮**，每轮 **1 题**；收答后再发下一题 |
| **批量 intake** | 用户**明确**说「一次问完」「可以问很多个」 | 一轮 **2–4 题**（见下通道限制） |

- 用户说「逐条拷问」「一个一个来」→ **顺序 grill**
- 用户说「可以问很多个」「一次问完」→ **批量 intake**（仍受 AskQuestion 题数上限约束）
- **默认一律顺序 grill**；含 ppt-maker 视觉 intake，**禁止**首轮一次塞 6 题

### Cursor `AskQuestion` 硬上限

| 规则 | 说明 |
|------|------|
| **每次调用 1 题** | `questions` 数组 **长度必须为 1** |
| **禁止** | 单次调用 2 题及以上（易报 `questions: Required` 或 UI 空题干） |
| **失败降级** | 报 `Invalid arguments` / `questions: Required` → 改 **单题重试**；仍失败 → **Markdown 选票（仅当前 1 题）** + 停止 |
| **批量 intake** | Cursor 上 **不得**用 AskQuestion 一次多题；若用户坚持一次问完 → **Markdown 选票** 列出 2–4 题 |

详见 [PLATFORMS.md](PLATFORMS.md) §Cursor 故障排除。

## 通道选择（跨 Agent 适配）

按当前宿主 **实际可用能力**，从高到低选一种通道（详见 [PLATFORMS.md](PLATFORMS.md)）：

| 优先级 | 通道 | 说明 |
|--------|------|------|
| **1** | **结构化选题工具** | 宿主提供的多选 / 表单类工具（如 Cursor `AskQuestion`） |
| **2** | **Markdown 选票** | 任意 Agent 均可用；无专用工具时的**标准降级**，不是次品 |

**规则：**

- 有结构化工具 → **必须用工具**，勿仅在正文列选项（否则无法稳定点选，Agent 易跳过等待）。
- 无结构化工具 → **必须用 Markdown 选票协议**（见下），并 **停止**等待回复。
- **禁止**因「当前是 Cursor / Claude Code / …」而跳过 intake；禁止把检索结果或 Agent 偏好当作用户选择。

## Markdown 选票协议（通用 fallback）

当宿主 **没有**结构化选题工具时使用：

```markdown
## 待你定稿 · [主题]

### 1. [问题 id] — [问题简述]（推荐：B — 理由一句）

- **A** · …
- **B** · …（推荐）
- **C** · …

### 2. …

请回复选项，例如：`1B, 2A` 或 `style_direction=B, color_mode=dark`。
**在你回复前我不会开始实现。**
```

要求：

- 每题 **2–6** 个互斥选项，字母或简短 `id` 均可
- 推荐项写在题头
- 末尾 **明确停止**；本轮 **不要** 调用写文件 / 执行变更类工具

## 工作流

```
1. 梳理待决项（决策树）
2. 剔除已由代码库/文档确认的事实
3. 默认「顺序 grill」；仅用户明确要求时用「批量 intake」
4. 按 PLATFORMS.md 选通道 → 每轮只发 1 题（AskQuestion）或 1 段选票
5. 等待用户答案 → 未闭合则下一题；全部闭合 → 表格复述共识
6. 已闭合 → 交付 / 开工
```

## 与联用技能

| 场景 | 联用 |
|------|------|
| PPT / Pitch 定风格 | **grill-me** 定视觉 → **ui-ux-pro-max** `--persist` |
| 计划抗压 + 成文 | **grill-me** → **grill-with-docs**（ADR / CONTEXT） |

## 反模式

- ❌ 正文里列了选项，但既没用结构化工具，也没用选票协议 + 停止等待
- ❌ 问卷发出后继续 scaffold / 改代码
- ❌ 把 skill 检索、黄金样例、Agent 偏好当作用户选择
- ❌ **AskQuestion 一次传入多题**（Cursor 已知不稳定）
- ❌ 工具报错后仍不降级、不重试单题
- ❌ 顺序依赖却一题塞 10 个无关维度
- ❌ 只有「是/否」、没有具名方案

## 完成标准

- 每个待决维度有 **用户显式选择**（工具返回、选票回复、或用户消息中的等价声明）
- Agent 已复述共识表格
- 无未声明默认假设进入实现
