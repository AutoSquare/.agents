# Git 提交说明与手动提交规范

## 触发条件

用户要求提交说明，或一次完整任务收尾需要建议提交时读取。

## 硬性约束

- 禁止 Agent 执行 `git add` / `git commit` / `git push`，除非用户在同一条消息里明确要求提交。
- Git 默认由用户本地手动提交；Agent 只生成可复制文案。
- 生成文案前只读检查 `git status --short`、`git diff --stat`，必要时读取近期提交风格。

## 文案风格

- 改动较小时用单行中文，以「修复了…」「优化了…」「新增了…」开头。
- 多模块联动时用：

```text
问题：<现象与后果>
处理：<做了什么；关键模块；必要验证>
```

## 禁止

- 不使用 Conventional Commits，如 `fix:`、`feat(scope):`。
- 不写 diff 中不存在的变更。
- 不把计划未落地项写进 message。
- 不提交密钥、`.env`、用户私有数据，除非用户明确要求。
