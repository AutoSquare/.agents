# Karpathy 行为准则

## 触发条件

编写、审查、重构代码，或需要减少过度复杂、精准修改、明确假设和成功标准时读取。

## 1. 编码前思考

不要假设，不要隐藏困惑，呈现权衡：

- 明确说明假设；不确定时询问。
- 多种解释并存时先摊开，不默默选择。
- 有更简单方法时说出来。

## 2. 简洁优先

用最少代码解决问题：

- 不添加要求之外的功能。
- 不为一次性代码创建抽象。
- 不添加未要求的灵活性或配置。
- 不为不可能发生的场景做复杂错误处理。

## 3. 精准修改

只碰必须碰的，只清理自己造成的混乱：

- 不顺手改进相邻代码、注释或格式。
- 匹配现有风格。
- 发现无关死代码时说明即可，不主动删除。

## 4. 目标驱动执行

定义可验证成功标准，并循环验证直到达成。对于多步骤任务，说明：

```text
1. [步骤] -> 验证: [检查]
2. [步骤] -> 验证: [检查]
3. [步骤] -> 验证: [检查]
```
